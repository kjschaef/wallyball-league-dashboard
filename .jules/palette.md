## 2024-03-24 - Segmented Controls Accessibility
**Learning:** Segmented control button groups (`role="tablist"`) can create a broken screen reader experience if they do not contain corresponding `role="tabpanel"` elements. In UI components like metrics toggles or season selectors, they function closer to a group of toggle buttons. Furthermore, relying on CSS borders without considering `border-radius` clamping can lead to visual clipping of `focus-visible` outlines.
**Action:** Replace `role="tablist"` with a standard container when using segmented controls as simple state toggles. Utilize native `<button>` elements with `aria-pressed={boolean}`. Ensure `focus-visible` styles include `z-10` to prevent clipping and explicitly define `rounded-l-md`/`rounded-r-md` on the edge buttons so the focus ring elegantly wraps the UI component.
